01 — Introduction

Project Overview

The Tours and Travel Management System is a complete, backend-driven web application that enables users to discover travel packages, make bookings, and process payments — all within a single, secure platform.

Administrators have a dedicated dashboard to manage packages, users, bookings, and payment records. The project is built following a clean layered architecture, demonstrating real-world patterns used in enterprise Spring Boot applications.

🌐
A static UI preview is deployed on Netlify for demonstration purposes: tours-and-travel-management-system.netlify.app. Backend features such as authentication, bookings, and payments require the local Spring Boot server to be running.
⚠️
This project was developed as part of a Master of Computer Applications (MCA) curriculum and is intended for educational and portfolio use.
02 — Design

System Architecture

The application follows a layered MVC architecture. Each layer has a single, clearly defined responsibility, promoting separation of concerns and testability.

Browser (HTTP Request) │ ▼ ┌─────────────────────────────────┐ │ Spring MVC Controller Layer │ ← Routes requests, handles HTTP └─────────────────┬───────────────┘ │ ▼ ┌─────────────────────────────────┐ │ Service Layer │ ← Business logic, orchestration └─────────────────┬───────────────┘ │ ▼ ┌─────────────────────────────────┐ │ Repository Layer (JPA) │ ← Data access, queries └─────────────────┬───────────────┘ │ ▼ ┌─────────────────────────────────┐ │ Database (H2 / MySQL) │ ← Persistence └─────────────────────────────────┘ │ ▼ Thymeleaf Templates → Rendered HTML → Browser

Key Architectural Decisions

DecisionRationale
Server-Side Rendering (Thymeleaf)Simplifies state management; no separate frontend framework required
Spring SecurityIndustry-standard authentication and role-based access control
H2 (dev) + MySQL (prod)Fast development iteration with H2; production-grade persistence with MySQL
Razorpay IntegrationPCI-compliant payment handling without storing card data on the server
Spring Data JPAReduced boilerplate; consistent repository pattern across all entities

05 — Flows

User Workflow

1

Register / Login

The user creates an account or logs in with existing credentials. Spring Security handles session management and password encryption.

2

Browse Travel Packages

The user navigates the packages listing page to explore available travel destinations and pricing.

3

Select a Package

The user views the full details of a chosen travel package, including the description and cost.

4

Confirm Booking

The user confirms their booking. A booking record is created in the database with a PENDING status.

5

Complete Payment

The user is redirected to the Razorpay payment widget to complete the transaction securely.

6

Booking Confirmed

Payment is verified by the backend. The booking status is updated to CONFIRMED and the user sees a confirmation page.

06 — Payments

Payment Flow

Payments are handled via the Razorpay Payment Gateway. The backend creates a Razorpay order, the frontend renders the payment widget, and the backend independently verifies the payment signature before confirming the booking.

User selects package & confirms booking │ ▼ Backend creates Razorpay Order (server-side) [razorpay.orders.create()] │ ▼ Frontend renders Razorpay checkout widget (order_id passed from backend to template) │ ▼ User completes payment (card / UPI / wallet) │ ▼ Razorpay returns: payment_id, order_id, signature │ ▼ Backend verifies HMAC-SHA256 signature [razorpay.utils.verifyPaymentSignature()] │ ┌────┴────┐ Valid Invalid │ │ ▼ ▼ Booking CONFIRMED Payment FAILED Payment stored Error shown to user
🔐
Payment verification is always performed server-side. The signature is validated using the Razorpay secret key before any booking is confirmed. Card details are never stored or processed by this application.

11 — Configuration

Developer Configuration

Razorpay Credentials

Add your Razorpay API credentials to src/main/resources/application.properties:

  application.properties 
# Razorpay Payment Gateway
razorpay.key_id=YOUR_RAZORPAY_KEY_ID
razorpay.key_secret=YOUR_RAZORPAY_KEY_SECRET
⚠️
Never commit your Razorpay secret key to a public repository. Add application.properties to .gitignore, or use environment variables to inject credentials securely.

MySQL Configuration (Production)

To switch from H2 to MySQL, update application.properties:

  application.properties 
# MySQL Datasource
spring.datasource.url=jdbc:mysql://localhost:3306/travel_db
spring.datasource.username=root
spring.datasource.password=yourpassword
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
12 — Security

Security Implementation

Authentication and authorization are implemented using Spring Security 6. Key aspects of the security implementation include:

FeatureImplementation
Password EncryptionBCryptPasswordEncoder — passwords are never stored in plain text
Session ManagementSpring Security session-based authentication with CSRF protection
Role-Based AccessTwo roles: ROLE_USER and ROLE_ADMIN
Route ProtectionAdmin routes secured via SecurityFilterChain configuration
Login / LogoutCustom login form and logout endpoint managed by Spring Security

Security Configuration (Conceptual)

  SecurityConfig.java 
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
          .authorizeHttpRequests(auth -> auth
            .requestMatchers("/admin/**").hasRole("ADMIN")
            .requestMatchers("/user/**").hasRole("USER")
            .requestMatchers("/", "/register", "/css/**").permitAll()
            .anyRequest().authenticated()
          )
          .formLogin(form -> form.loginPage("/login").permitAll())
          .logout(logout -> logout.permitAll());
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
